This Application

This Spring Boot application showcases production-grade cloud-native development practices. It demonstrates end-to-end automation from code commit to production deployment, leveraging modern DevOps tools and AWS infrastructure. The application implements industry best practices for containerization, orchestration, monitoring, and continuous delivery.

Architecture

  • The application runs on Amazon EKS (Elastic Kubernetes Service), providing scalable container orchestration with high availability across multiple availability zones.
  • Traffic is routed through an AWS Application Load Balancer, which provides path-based routing to both the application and Grafana monitoring dashboard.
  • GitHub Actions runners execute directly within the Kubernetes cluster, enabling secure and efficient CI/CD workflows without external dependencies.
  • Docker-in-Docker containers allow for isolated, containerized builds within Kubernetes pods, ensuring build environment consistency.
  • Multi-stage Docker builds optimize image size and security by separating build dependencies from runtime requirements.
  • Spring Boot Actuator exposes Prometheus-compatible metrics at /actuator/prometheus, enabling comprehensive application monitoring.
  • Grafana dashboards provide real-time visualization of application performance, resource utilization, and system health metrics.

Monitoring & Observability

  • Spring Boot Actuator provides production-ready features including health checks, metrics collection, and application insights.
  • Prometheus scrapes metrics every 30 seconds, storing time-series data for historical analysis and alerting.
  • Grafana offers customizable dashboards for visualizing application metrics, infrastructure health, and business KPIs.
  • The monitoring stack enables proactive issue detection and performance optimization through real-time alerting and trend analysis.
